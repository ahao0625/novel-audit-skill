# 审计标准参考文档

本文档为 novel-audit 技能的核心判断依据，定义11个审计维度的详细标准、典型反例和判断方法。

---

## 一、A类：文本质量类

### A1 文字基础质量

**定义：** 检测文本在语言表达层面的基础质量，包括语法、用词、句式、描写深度。

**检测项目及判断标准：**

#### 1.1 病句与语法错误
- **成分残缺**：缺主语、缺谓语、缺宾语
  - ❌ 反例："随着科技的发展，使人们的生活越来越便利。"（"随着…"后主语缺失）
  - ✅ 修正："随着科技的发展，人们的生活越来越便利。"
- **搭配不当**：主谓/动宾/修饰语搭配错误
  - ❌ 反例："他的眼神散发出深邃的光芒。"（眼神不能"散发光芒"）
  - ✅ 修正："他的目光深邃，像深潭一般。"
- **逻辑混乱**：句子内部因果/转折关系矛盾
  - ❌ 反例："虽然他很努力，所以他成功了。"（应用"但是"而非"所以"）

#### 1.2 流水账写法
**判定标准**：连续3句以上均为"主语+动词+宾语"的简单陈述，无感官描写、无情绪描写、无细节。

❌ 典型流水账：
```
他走进房间。他看了看桌子。他坐了下来。他拿起了书。他开始看书。
```

✅ 优质改写：
```
房间里很安静，只有窗外偶尔传来的鸟鸣。他推开门，目光扫过凌乱的桌面，在椅子上坐下，随手拿起最近的那本书——并不是因为想看，只是因为不知道该做什么。
```

**流水账判定触发条件**：
- 连续5句动作句（动词为主，无修饰）
- 段落内无任何感官描写（视/听/嗅/触/味）
- 没有人物内心活动或情绪反应

#### 1.3 形容词堆砌
**判定标准**：同一处描写连用3个以上形容词，且语义重叠或层次不清。

❌ 反例："她有着美丽动人、光彩照人、令人心动的容颜。"
✅ 修正："她的眉眼生得极好，只是随意一笑，就让人挪不开目光。"

#### 1.4 冗余表达
**判定标准**：删掉某些词/句后，意思完全不变。

❌ 反例："他心里在内心深处想着"（"心里"和"在内心深处"重复）
❌ 反例："已经完成了的任务"→"已完成的任务"

#### 1.5 错别字与不规范用词
- 常见错别字：暴（爆）发、座（坐）落、竟（竟）然
- 不规范网络用词（根据上下文判断是否为故意为之）

### A2 重复雷同检测

**定义：** 检测文本中存在的重复内容，包括词语重复、句式重复、情节结构重复。

#### 2.1 词语高频重复
**判定标准**：同一个词在500字内出现3次以上（人名、专有名词除外）。

**检测方法**：
- 扫描高频动词：如"看"、"走"、"说"等基础动词出现过多
- 扫描高频形容词：如"淡淡的"、"微微"、"轻轻"等习惯性修饰词
- 扫描重复短语：如"深吸一口气"、"眸子微动"等惯用表达

#### 2.2 段内同义重复
**判定标准**：同一段落内，两句话表达同一个意思。

❌ 反例：
```
他感到非常愤怒，心中的怒火难以平息。他愤恨地看着对方，充满了怒意。
```
（"愤怒"、"怒火"、"愤恨"、"怒意"在一段内出现4次）

#### 2.3 情节结构克隆
**判定标准**：两个场景/段落的叙事结构高度相似，即"起因-经过-反应"的框架几乎一致。

典型克隆结构（常见于AI生成）：
```
场景A：主角遇到X → X提出挑战 → 主角接受 → 主角轻松赢了 → 旁观者惊叹
场景B：主角遇到Y → Y提出挑战 → 主角接受 → 主角轻松赢了 → 旁观者惊叹
```

#### 2.4 跨章重复
**多章模式下检测**：
- 前章已解释的背景，本章重复大段复述
- 前章已出现的场景描写，本章几乎原文照搬
- 人物的"性格展示"场景前后雷同

### A3 AI痕迹识别

**定义：** 检测文本中具有AI生成特征的语言模式，并提供去AI化改写建议。

**重要说明：** AI痕迹判断需结合上下文，不能因为"排比句"就判定为AI生成。判断依据是"多个AI特征同时出现"。

#### 3.1 高频AI特征句式（详细词库见 ai-patterns.md）

**类型1：总结性内心独白**
AI常用主角视角来总结刚发生的事，语气像旁白，而非人物真实思维。
❌ AI味："他深知，这场考验才刚刚开始。"
❌ AI味："她明白，命运的齿轮已经开始转动。"
✅ 人味："他站在那里，脑子里乱糟糟的，只是觉得——还没完。"

**类型2：开篇环境描写套路**
❌ AI味："阳光透过窗户洒落，空气中弥漫着清晨的气息，一切都显得那么宁静美好。"
✅ 人味：用一个具体的、异常的细节切入，而不是全景式晨光描写。

**类型3：情感表达公式化**
❌ AI味："他的心中涌现出一股莫名的感动，眼眶微微湿润了。"
✅ 人味：用行为/细节表达情绪，不直接点名情绪。

**类型4：过度对称排比**
❌ AI味："有人说，人生是一场旅途；有人说，人生是一本书；有人说，人生是一幕戏。"
（真实写作中排比不会这么工整且连续）

**类型5："正当此时"式情节推进**
❌ AI味："正当他陷入沉思之际，突然，一阵敲门声响起。"
❌ AI味："就在这千钧一发之际，一道声音从远处传来。"
（AI特别爱用这类"打断沉默"的过渡）

**类型6：旁观者惊叹模板**
❌ AI味："周围的人都惊呆了，他们没想到眼前这个少年竟然如此强大。"
❌ AI味："在场所有人都倒吸一口冷气，面面相觑。"

**类型7：万能感叹句收尾**
❌ AI味："这一切，都只是刚刚开始……"
❌ AI味："他望着远处，心中百感交集。"
（AI极爱用省略号+情绪泛指词收尾）

#### 3.2 AI痕迹综合判定标准

| AI特征出现数量 | 判定结果 | 评分参考 |
|---|---|---|
| 0-1个 | 基本无AI痕迹 | 9-10分 |
| 2-3个 | 轻微AI痕迹 | 7-8分 |
| 4-5个 | 中度AI痕迹，需改写 | 5-6分 |
| 6个以上 | 重度AI痕迹 | 1-4分 |

#### 3.3 去AI化改写原则
1. **用细节替换抽象**：把"他感到很感动"改成一个具体的细节动作
2. **打破对称**：把整齐的排比句改成长短不一、参差错落的表达
3. **用口语/方言替换书面套话**：让人物说话像真人，而不是朗读
4. **删掉旁观者惊叹段落**：让读者自己感受强大，而不是由旁观者告诉读者
5. **结尾不用省略号+泛情绪**：改成一个具体的行动或细节来收尾

## 二、B类：叙事逻辑类

### B1 情节合理性

**定义：** 检测情节发展是否符合逻辑，人物行为是否有合理动机，关键转折是否有充分铺垫。

#### 检测点1：动机充分性
**判定标准**：每个重要行为必须有前文可追溯的动机。

缺乏动机的典型情况：
- 人物突然做出重大决定，但前文无任何铺垫
- 反派无缘无故出现/消失
- 人物突然对另一人物产生好感/恶感，无触发事件

**检测方法**：对文本中每个"重要行为"，往前追溯动机：
- 这个行为为什么发生？（直接原因）
- 是什么让这个角色选择了这个行动而不是别的？（深层动机）

#### 检测点2：因果链完整性
**判定标准**：A导致B，B导致C，因果链不能有跳跃。

❌ 反例（因果跳跃）：
```
第一段：他拒绝了长老的要求。
第三段：他成为了宗门新的掌门。
```
（中间缺少"他如何通过了考验/如何赢得了支持"的过程）

#### 检测点3：上帝之手
**定义**：明显是作者强行让情节朝预设方向发展，不符合故事内在逻辑。

典型表现：
- 强大的反派突然犯蠢，给主角可乘之机
- 巧合过多（主角刚好每次都遇到对的人/得到对的东西）
- 危险突然消失，没有合理解释

#### 检测点4：铺垫与兑现
**判定标准**：文本中有没有前文已埋的伏笔在本章被合理兑现？

- 检查是否有"无铺垫的高潮"：突然发生的大事件，前文毫无预兆
- 检查是否有"铺而不发"：前文大量铺垫，但本章完全没有推进

---

### B2 时间线一致性

**定义：** 检测文本中的时间信息是否前后一致，包括事件顺序、人物年龄、季节天气等。

#### 检测点1：事件时序矛盾
- 标记文本中所有时间词："昨天"、"上个月"、"三年前"、"子时"等
- 验证时间词之间的关系是否自洽

❌ 典型矛盾：
```
第二段：他三年前离开了家乡。
第五段：两年前，他刚刚告别了父母。
```
（同一件事，时间描述矛盾）

#### 检测点2：年龄/成长一致性
- 文中提及的年龄与经历是否匹配
- 人物完成某件事所需的时间是否合理

#### 检测点3：季节与自然环境
- 季节描写在同一时间段内是否一致
- ❌ 反例：开篇说"深冬大雪"，几段后写"柳絮飘飞"（同一天发生，但季节矛盾）

#### 检测点4：故事内时钟
- 事件发展的速度是否合理
- ❌ 反例：早上出发，中午到达，但距离被描述为"千里之遥"

---

### B3 上下文连贯性

**定义：** 检测本章文本与前文（或章内段落之间）在设定、人物状态、情绪氛围上的连贯性。

#### 检测点1：设定连贯性
- 前文已确立的世界规则/道具/地点，在本章是否被正确延续
- ❌ 反例：前章说"这把剑只有真正的传承者才能拔出"，本章任何人都随手拿起了它

#### 检测点2：人物状态连贯性
- 上章/上段结尾时角色处于何种状态（受伤/情绪/位置），本章开头是否合理延续
- ❌ 反例：上章主角重伤昏迷，本章开头跑步训练，无任何时间跳跃交代

#### 检测点3：情绪承接
- 重要情绪事件发生后，人物的情绪变化是否有合理的延续
- ❌ 反例：亲人刚死，下一段主角就在开玩笑，无任何过渡

#### 检测点4：信息连贯性
- 某个角色在前文明确知道的信息，不应在后文表现得"毫不知情"
- ❌ 反例：前文主角偷听到了密谋，但后文见到反派时完全无防备

---

### B4 叙事视角一致性

**定义：** 检测叙事视角（POV）是否始终保持一致，有无视角跳跃或信息越界。

#### 检测点1：视角越界（限知视角）

**判定标准**：在第三人称限知/第一人称叙事中，写出视角人物不可能知道的信息。

❌ 反例（第三人称限知跟随李明）：
```
李明推开门，房间里没有人。
他不知道王强此刻正在三楼的暗室里准备着陷阱。
```
（李明不可能知道王强在三楼做什么——视角越界）

✅ 修正：
```
李明推开门，房间里没有人。
太安静了。安静得让人不放心。
```

#### 检测点2：读取他人内心

**判定标准**：视角人物直接获知另一角色的内心活动（而非通过观察推测）。

❌ 反例：
```
李明看着她，她心里很害怕，但她努力掩饰着。
```
（视角是李明，他只能"看到她努力掩饰"，不能知道"她心里害怕"）

✅ 修正：
```
李明看着她。
她脸上没什么表情，但手指在桌子下面攥得很紧。
```

#### 检测点3：无预告的视角切换

**判定标准**：在未用分节线（`---`）或空行明确分隔的情况下，直接切换视角人物。

**优化方案**：
- 如果是有意切换视角，在切换前加分节线或空行明确分隔
- 如果是无意切换，统一回主视角人物

## 三、C类：读者体验类

### C1 节奏与爽点设计

**定义：** 检测文本的叙事节奏是否合理，是否有足够的情绪高点和读者粘性设计。

#### 检测点1：拖沓识别
**判定标准**：以下情况视为节奏拖沓——
- 连续500字以上无任何情节推进（纯描写/内心独白/反复铺垫）
- 同一个情绪/场景反复描写，超过3次仍未推进
- 对话中大量无信息量的寒暄（"你好吗""还好还好"这类）

#### 检测点2：爽点密度
**参考标准**（按题材不同有所差异）：
- 爽文/爽点型网文：每1000-2000字应有至少1个小爽点
- 情感/慢热型：爽点可以更稀疏，但情绪积累要扎实
- 判断爽点是否"够爽"：旁观者是否有强烈反应？主角是否有明确的成就感？

#### 检测点3：章末钩子
- 优秀钩子：留下一个未解答的问题、或刚发生了意外转折
- 及格钩子：章末有明确悬念
- 不及格：章末完全收束，无任何后续期待

#### 检测点4：情绪曲线
- 理想情绪曲线：低开→积累→高潮→短暂释放→新的悬念
- 问题识别：一直高亢（情绪疲劳）、一直平淡（无代入感）、高潮来得太突然（无积累）

#### 检测点5：开篇吸引力（黄金开篇原则）

**判定标准**：开篇1000字内是否满足以下要素——

- **切入是否快速**：第1段是否直接进入事件/情境，而非大段背景介绍
- **代入感建立**：前500字内是否有主角的明确目标/困境/情感状态
- **金手指/主线是否尽早展示**：对于爽文/系统文，第3章内应该出现核心设定

❌ 问题开篇（背景堆砌）：
```
大周王朝，永宁三十七年，天下三分，百姓疾苦……
（300字背景介绍后，主角才出场）
```

✅ 优质开篇（快速切入）：
```
林默醒来时，嘴里全是血。
牢房的门"吱呀"一声开了，一个狱卒探进头来："今日午时，问斩。"
他想坐起来，才发现手脚都锁着铁链。
```

#### 检测点6：信息密度

**判定标准**：信息密度过高（读者看不懂）和过低（流水账）都需预警。

- **密度过低**：连续200字以上只有动作罗列，无感官/情绪/环境信息 → 参见A1流水账检测
- **密度过高**：连续200字以上密集投放新设定/人名/专有名词，读者难以消化
- **Show vs Tell失衡**：
  - Tell过多（"他觉得很生气"）→ 应改为展示细节
  - Show过多（大段环境描写无情节推进）→ 应压缩或穿插情节

---

### C2 人物塑造一致性

**定义：** 检测主要角色在本文本中的性格、行为、对话是否与其设定一致。

#### 检测点1：性格一致性
- 为主要角色建立一个简短的性格标签（根据文本内容）
- 验证每个重要场景中，角色的反应是否符合该性格标签

典型性格矛盾：
- 设定为"冷静沉稳"的角色，遇到小事就情绪失控
- 设定为"刚正不阿"的角色，为了利益突然妥协（无铺垫）

#### 检测点2：对话口吻
每个角色的说话方式应该有辨识度：
- 受教育程度体现在用词层次
- 性格体现在句子结构（话少/话多、直接/迂回）
- 身份/地位体现在称谓和语气

❌ 反例：一个农村老农说出了书面感极强的文绉绉的长句。

#### 检测点3：成长弧合理性
- 如果角色在本章有明显的心理变化/成长，检查是否有足够的触发事件支撑
- 变化不能太快（"性格转变"需要积累）也不能太慢（"毫无变化"让人觉得人物平面）

---

### C3 世界观自洽性

**定义：** 检测文本的世界规则体系是否前后一致，不同题材有不同的检测重点。

#### 通用检测点

**规则一致性：**
- 文本中有没有"打破规则"但没有给出合理代价的情况
- ❌ 反例：设定"使用魔法必须消耗生命力"，但主角大量使用魔法毫无代价

**内部逻辑：**
- 强大的力量是否有对应的限制/代价
- 世界中的势力关系是否符合基本权力逻辑

---

### C4 对话质量专项

**定义：** 检测对话是否具有人物辨识度、信息密度是否合理、是否兼具情节推进和人物塑造功能。

#### 检测点1：对话同质化

**判定标准**：去掉角色名后，无法判断是谁在说话。

❌ 同质化对话（所有人说话方式相同）：
```
"我觉得这件事不太对劲。"李明说。
"我也这么认为。"王强说。
"那我们该怎么办？"赵刚问。
```

✅ 区分对话DNA后：
```
"不对劲。"李明就说了两个字。
王强想了想："……嗯。"
赵刚急了："那怎么办啊？要不先跑吧？"
```

**优化方案**：为每个主要角色建立"对话DNA"（见 `references/optimization-guide.md` 对话专项部分）。

#### 检测点2：对话功能化（只传递信息）

**判定标准**：对话内容100%是信息传递，没有废话、没有情绪、没有弦外之音。

**优化方案**：在信息型对话中插入人类真实对话的"噪音"——
- 打断/抢话
- 答非所问
- 话题偏移
- 重复对方的话（表示在思考）
- 无意义的语气词

#### 检测点3：对话信息密度过高

**判定标准**：单个人物连续说超过3句话，且每句都在传递新信息。

❌ 信息过载：
```
"明天行动时间是早上六点，地点在城南废弃工厂，记住不要带手机，到了之后先观察周围情况再进去。"
```

✅ 用动作/表情隔开：
```
"明天，六点。"他停了一下，"城南那个废工厂。"
"……六点？那不是天还没亮？"
"就是要天没亮。"
她没再问，把地图折起来塞进口袋。
```

#### 检测点4：弦外之音缺失

**判定标准**：所有对话都是直白表达，无人物的隐藏动机、无人物的言外之意。

**优化方案**：让至少30%的对话含有弦外之音——人物说的和想的不完全一致。

## 四、题材专项检查点

### 玄幻/仙侠/修真类

**额外检测项：**
- **战力体系自洽**：境界/等级的描述是否前后一致？敌我实力差距是否符合设定？
- **升级节奏**：升级速度是否合理？突破是否有铺垫（修炼时间/机缘）？
- **打脸是否合理**：反派的"蠢"是否有逻辑依据，还是纯粹为了让主角打脸？
- **系统/金手指**：系统奖励是否与情节强度匹配？有没有"无缘无故掉大宝贝"？
- **宗门/势力关系**：派系关系是否前后一致？

### 都市/现实类

**额外检测项：**
- **现实感**：是否有明显不符合现实的常识错误（法律/医学/职场惯例）？
- **人际关系逻辑**：人物的社会关系网络是否合理？
- **金钱/资产描写**：数字是否符合现实比例？
- **情感线推进**：感情发展速度是否合理？有没有"闪恋"缺乏积累？

### 悬疑/推理类

**额外检测项：**
- **线索投放**：核心线索是否在解谜前已经投放？（不能临时出现新线索）
- **逻辑自洽**：推理过程是否严密？结论与证据是否匹配？
- **公平原则**：读者是否有足够信息与主角一起推理？

### 言情/情感类

**额外检测项：**
- **感情积累**：感情发展是否有足够的互动积累？
- **误会设计**：误会是否合理？当事人是否有机会澄清但刻意回避？
- **BE/HE合理性**：结局走向是否有足够铺垫？

### 历史/架空历史类

**额外检测项：**
- **历史氛围**：用词/称谓/礼仪是否符合时代背景？
- **权谋逻辑**：政治斗争的逻辑是否合理？势力关系是否符合历史规律？
- **地理正确**：地名/地理描写是否与时代设定匹配？
