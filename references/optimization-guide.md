# 网文优化方案参考指南

本文档为审计技能提供各维度问题的具体优化方案和改写示例。
审计时发现对应问题，应参考本文档给出具体改写建议。

---

## 使用说明

- 每个维度下列出「常见问题类型 → 优化方案 → 改写示例」
- 改写示例格式：`❌ 问题原文` → `✅ 优化后` + `改写思路`
- 优化方案应具体到"可以怎么改"，而不是泛泛说"建议加强描写"

## A1 文字基础质量 — 优化方案

### 问题类型一：语病/标点错误

| 问题 | 优化方案 | 示例 |
|---|---|---|
| 主语缺失 | 补充主语或调整句式 | ❌ "走进了房间，环顾四周。" → ✅ "他走进了房间，环顾四周。" |
| 标点误用（逗号句号混用） | 按语义断句，该句号的地方用句号 | ❌ "他看了看天，要下雨了。" → ✅ "他看了看天。要下雨了。" |
| "的/地/得"混用 | 的+名词，地+动词，得+补语 | ❌ "他跑的很快" → ✅ "他跑得很快" |

### 问题类型二：流水账（动作堆砌无描写）

**识别特征：** 连续3句以上只有"谁做了什么"，无感官描写、无心理、无环境。

**优化方案：**
在动作之间插入以下任意一种：
- 一个感官细节（看到的/听到的/闻到的）
- 一句心理活动（想法/情绪）
- 一处环境互动（风/光/温度对角色的影响）

**改写示例：**

❌ 问题原文：
```
张明站了起来。他拿起外套。他走出了办公室。他上了车。
```

✅ 优化后（方案A：加感官细节）：
```
张明站了起来，膝盖有点酸——坐太久了。
他拿起外套，闻到一股淡淡的烟草味，是昨天在老家沾上的。
走出办公室的时候，走廊的灯闪了一下。
他上了车，发动引擎，暖气还没来。
```

✅ 优化后（方案B：加心理活动）：
```
张明站了起来，决定不再想这件事。
他拿起外套，走出了办公室。
走廊尽头，同事小李正朝他挥手，他假装没看见，加快了脚步。
```

### 问题类型三：冗余表达

**常见冗余模式：**

| 冗余模式 | 优化方案 |
|---|---|
| "他心里想/他脑海中/他在心里" + 想法 | 直接写想法，删掉引导语 |
| "亲眼看到/亲耳听到" | 删掉"亲眼/亲耳" |
| "毫无疑问/显而易见/可想而知" | 删掉，让内容自己说明 |
| 连续两句意思几乎相同 | 合并为一句，保留更具体的那句 |

**改写示例：**

❌ `他心里想：这件事不能就这么算了。`
✅ `这件事不能就这么算了。`

❌ `他亲眼看到了那一幕，毫无疑问，是她。`
✅ `他看到了那一幕，是她。`

## A2 重复雷同 — 优化方案

### 问题类型一：高频词泛滥

**优化方案：** 用同义词替换 + 改变表达方式

**常见高频词替换表：**

| 原词 | 替换方案 |
|---|---|
| 突然 | 猛地 / 毫无预兆地 / 还没反应过来 / 一瞬间 |
| 淡淡地 | 语气很轻 / 像在自言自语 / 没什么表情地说 |
| 眼中闪过 | 眼神变了 / 目光一沉 / 眼皮抬了一下 |
| 深吸一口气 | 屏住呼吸三秒 / 把一口气慢慢吐出来 / 胸口起伏了一下 |
| 就在这时 | 话音刚落 / 脚步声从身后传来 / 手机突然震了 |
| 不得不承认 | 没法反驳 / 事实摆在面前 / 再嘴硬也没用 |

### 问题类型二：段落结构克隆

**识别特征：** 两段用完全相同的句式结构，只是换了人名和道具。

**优化方案：** 打散结构，让两段用不同的节奏和切入角度。

**改写示例：**

❌ 问题原文（两段结构雷同）：
```
李明拔出长剑，剑身上泛起淡淡的光芒。他一剑斩出，空气都被撕裂了。
王强举起大锤，锤头上冒着熊熊的火焰。他一锤砸下，地面都震动了。
```

✅ 优化后：
```
李明拔出长剑。
剑身泛起微光，不是很亮，但足够冷。
他没说话，一剑斩出——

王强那边动静大多了。
大锤举起的时候，火焰裹住了整个锤头，热浪扑面。
他砸下的那一秒，地面像鼓面一样震动起来。
```

## A3 AI痕迹 — 优化方案

> 详见 `ai-patterns.md` 第四节「去AI化改写策略」，以下为补充示例。

### 问题类型一：情绪量化套话

| AI套话 | 去AI化方案 |
|---|---|
| "心中涌现出一股..." | 写一个微小动作（手抖/鼻子酸/低下头） |
| "眼眶湿润" | 写他怎么掩盖（转过身/抬头看天/假装揉眼睛） |
| "百感交集" | 删掉，写两个具体念头快速切换 |

**改写示例：**

❌ `看着母亲的背影，他心中涌现出一股说不出的感动，眼眶湿润了。`
✅ `看着母亲转身出去，他站在原地，好一会儿才低头揉了揉鼻子。`

### 问题类型二：旁观者惊叹模板

**优化方案：** 删掉群体反应，改成一两个具体人物的具体动作。

❌ `在场所有人都惊呆了，没有人说话。`
✅ `最前面那个年轻弟子手里的剑掉了，"咣"一声砸在地上。`

## B1 情节合理性 — 优化方案

### 问题类型一：动机缺失

**优化方案：** 在行为前插入动机铺垫（一句话即可）

**改写示例：**

❌ 问题原文：
```
张明突然一拳砸在桌上。"不干了！"
```

✅ 优化后（补充动机）：
```
张明盯着那份文件看了三遍，每一遍都更生气。
他忽然一拳砸在桌上，纸张跳了起来。
"不干了！"
```

### 问题类型二：因果链断裂

**优化方案：** 补入中间步骤，让"因为A所以B"变成"因为A→（中间事件）→所以B"

**改写示例：**

❌ `他练了三天，终于突破了。`（因果过于简略，像过关）
✅ `他练了三天，每天只睡两个时辰。第四天清晨，灵气在经脉里走了第一个完整的循环——他知道，成了。`

### 问题类型三：上帝之手（作者强行推动情节）

**识别特征：** 关键转折靠巧合、路人突然送来关键信息、反派突然降智。

**优化方案：** 让转折来自角色的选择或已有伏笔，而不是巧合。

**改写示例：**

❌ `正当他一筹莫展时，一个陌生人走过来，给了他一本秘籍。`（上帝之手）
✅ `他翻遍了师父的遗物，终于在书箱夹层里摸到一张泛黄的纸条——那是师父临终前塞进去的，他当时没注意到。`

## B2 时间线一致性 — 优化方案

### 问题类型一：时序矛盾

**优化方案：** 明确时间锚点，用具体时间词（凌晨/清晨/午后/黄昏/夜深）替代模糊表达。

**改写示例：**

❌ 问题原文：
```
昨天他去了县城。今天早上，他想起来前天忘记关门了。
```
（"前天"在时间线上混乱——昨天去县城，前天是去县城之前的两天）

✅ 优化后：
```
昨天他去了县城。
今晚躺在旅馆的床上，他突然想起——出门前，他好像没关厨房的门。
```

### 问题类型二：时间流速不一致

**识别特征：** 同一段时间内，不同角色的经历时长感矛盾。

**优化方案：** 用具体的时段描写（一炷香/半盏茶/一整天/三天三夜）来统一时间感。

## B3 上下文连贯性 — 优化方案

### 问题类型一：情绪/态度突变无过渡

**优化方案：** 在情绪转换处加入"触发事件"或"心理过渡"。

**改写示例：**

❌ 问题原文：
```
"我恨你。"她咬着牙说。
"那好吧。"他忽然笑了，眼神温柔。
```
（从"恨"到"温柔"完全无过渡）

✅ 优化后：
```
"我恨你。"她咬着牙说。
他沉默了好一会儿，忽然笑了。
那笑容很轻，像是认了，也像是放弃了什么。
"那好吧。"
```

### 问题类型二：设定遗忘

**优化方案：** 在章节开头或关键节点，用角色行动"唤醒"设定，而不是直接说明。

**改写示例：**

设定：主角左手残疾。
❌ 问题：后续章节主角用左手拿杯子。
✅ 修复：要么让主角用右手拿杯子，要么写"他用左手——那只以为再也握不住剑的手——紧紧握住了杯子"，把设定融入行动。

## C1 节奏与爽点 — 优化方案

### 问题类型一：拖沓段落（无实质推进）

**优化方案：** 删掉"过渡性废话"，保留推进情节或塑造人物的内容。

**删减优先级：**
1. 角色走路/坐车的过程描写（除非有重要对话发生）
2. 无信息量的环境描写（天气/风景，除非与情节相关）
3. 重复表达同一情绪的多句话

**改写示例：**

❌ 问题原文（拖沓）：
```
李明走出了门。
外面的阳光很刺眼，他眯了眯眼睛。
他沿着小路往前走，路边的野花开得正好。
走了大约十分钟，他来到了镇上。
```
（四句话，信息量约等于零）

✅ 优化后：
```
李明出门的时候，阳光刺得他眯了眼。
等他走到镇上，那股刺目的感觉还在眼皮上跳。
```

### 问题类型二：章末无钩子

**优化方案：** 在章末最后一段加入以下任意一种：
- 一个出乎意料的消息/发现
- 一句意味深长的话（谁来写的信/谁留下的痕迹）
- 主角的行动计划被突然打断
- 视角切换到反派/另一角色，揭示新信息

**改写示例：**

❌ 问题原文（平淡结尾）：
```
张明回到了家，累得倒头就睡。
```

✅ 优化后（加钩子）：
```
张明回到家的时侯，灯是亮的。
他记得自己出门前关了灯。
桌上放着一杯还冒着热气的茶。
```

## C2 人物塑造 — 优化方案

### 问题类型一：性格前后矛盾

**优化方案：** 如果性格变化是有意的，需要铺垫"触发事件"；如果是无意的矛盾，统一到其中一种。

**改写示例：**

设定：角色A性格冷静理智。
❌ 问题：某章角色A突然冲动地大喊大叫，无铺垫。
✅ 修复方案A（有意变化，加铺垫）：
```
这几天发生的事情，像石头一样压在他胸口。
他一直忍着，一直算着，一直告诉自己冷静。
然后，某个人说了一句话。
他终于忍不了了。
```
✅ 修复方案B（回归设定）：
把冲动行为改成"他手指微微发抖，但声音很平静地说……"，用细节表现克制。

### 问题类型二：对话口吻不符身份

**优化方案：** 根据身份调整用词、句式长度、语气词。

| 身份 | 口吻特征 |
|---|---|
| 古代文人 | 文言词/对仗句/引经据典，少用"吧/啊/哦" |
| 现代都市年轻人 | 短句/网络用语/省略主语，语气词丰富 |
| 反派大佬 | 句子短而有压迫感，少解释，多命令式 |
| 配角路人 | 功能性强，话说完就退场，不抢戏 |

## C3 世界观自洽性 — 优化方案

### 问题类型一：规则矛盾（设定被打脸）

**优化方案：** 如果后续情节需要打破规则，必须给出解释（代价/例外/隐藏设定）。

**改写示例：**

设定：修真界"金丹期以上不能进入凡人城池"。
❌ 问题：某章金丹期高手直接进了城，无解释。
✅ 修复：
```
他知道规矩。
但今天，他顾不上规矩了。
腰间的宗门令牌闪了一下——这是他最后一次动用长老权限，后果他之后再去承担。
```

### 问题类型二：数值/等级不自洽

**优化方案：** 建立简单的"数值记录"，在长文写作时定期核对。

建议作者在正文外的笔记中维护：
```
角色名 | 当前等级/战力 | 关键装备 | 上次更新章节
```

## 综合优化原则

1. **展示，不要告诉（Show, Don't Tell）**
   - ❌ "他很生气" → ✅ "他把杯子摔在了地上"

2. **用动作代替情绪形容词**
   - ❌ "她很紧张" → ✅ "她反复摸着口袋里的打火机"

3. **对话要有"弦外之音"**
   - ❌ 人物把想法全说出来 → ✅ 人物说一半，留一半

4. **每段都要有"存在理由"**
   - 推进情节 / 塑造人物 / 埋伏笔 —— 至少要占一样

5. **章末最后一句，要么留悬念，要么留余味**
   - 不要以"他睡着了""就这样结束了"这类封闭式句子结尾

## 去AI化主动优化六法（2026年5月补充）

以下六法借鉴自 qu-ai-wei 项目的 craft moves，适配网文写作场景。
审计时除了"去掉AI痕迹"，更应主动建议这些正面优化方向。

### 方法一：动词强化

**原则**：AI爱用抽象万能动词（进行/开展/实现/发挥/推动），真人用具体动作动词。

| AI抽象动词 | 具体动作替代 | 网文场景示例 |
|---|---|---|
| 进行修炼 | 打坐/闭关/练功 | ❌ "他进行了深入修炼" → ✅ "他盘膝坐下，灵气沿经脉运转了三个大周天" |
| 实现突破 | 突破了/冲过去了 | ❌ "他终于实现了突破" → ✅ "丹田里'轰'的一声，像什么炸开了" |
| 发挥实力 | 出手了/动真格了 | ❌ "他发挥了自己的实力" → ✅ "他终于不再藏着掖着，第一拳就砸碎了石柱" |
| 推动发展/变革 | 做成了/搞定了 | 不适用于叙事，改写时直接删掉 |
| 赋能/激发 | 帮了/逼出了/催动了 | 不适用于叙事，改写时改为具体因果 |

**审计建议**：扫描全文"进行/开展/实施/实现/发挥/推动/打造/构建"等词，逐一替换为具体动作。

### 方法二：节奏重塑

**原则**：打破AI的"机关枪节奏"——连续5句以上长度相近（15-30字）是典型AI特征。

**操作方法**：
- **合并过碎的短句**：把2-3个语义相关的短句合为一句
- **拆开过长的从句**：在关键位置用句号切断
- **插入短句制造节奏变化**：在长句之间塞一句3-5字的超短句

❌ 节奏均质化（每句都在20字左右）：
```
他缓缓地走出了房间，外面的阳光非常刺眼。
走廊里空无一人，只有他的脚步声在回响。
他来到了一楼大厅，看到了那个熟悉的人影。
```

✅ 节奏重塑后：
```
他推开门，阳光刺得眯了眼。
走廊里没人。脚步声一下一下的。
一楼大厅——那个人还在。
```

### 方法三：Filler切除

**原则**：删掉不承载信息的"包装词"，让叙事更紧凑。

**常见目标词（网文场景）：**

| Filler 类型 | 示例 | 处理方式 |
|---|---|---|
| 模糊限定 | "在一定程度上""某种意义上""可能""也许" | 直接删掉 |
| 冗余引导 | "心里想着""在脑海中""他知道" | 删引导语，直接写想法 |
| 空转强调 | "值得注意的是""不得不说""必须承认" | 直接删掉 |
| 过度修饰 | "一些""各种""相关""各个方面" | 能删就删 |

❌ `他心里想着，在这种情况下，他不得不承认，某种意义上自己确实害怕了。`
✅ `他确实怕了。`

### 方法四：抽象换具体

**原则**：把空洞的泛化描写换成原文中已有的具体细节（禁止凭空编造新细节）。

| 维度 | 抽象写法 | 具体写法 |
|---|---|---|
| 场景 | "繁华的街道" | "街上卖糖葫芦的吆喝声盖过了车马声" |
| 人物 | "她很美" | "她把头发绾上去的时候，后颈露出一小片白" |
| 武力 | "实力强大" | "他一掌拍下，青石地砖裂了八块" |
| 情绪 | "他很感动" | "他把那封信叠好，放进贴身的口袋里" |

**重要约束**：只能使用原文中已给出的信息，不得凭空发明新细节。

### 方法五：语序归位（反翻译腔）

**原则**：修复AI文本中残留的英文语法结构，让中文回归自然语序。

| 翻译腔表现 | 修正方式 |
|---|---|
| 代词泛滥（"他拿起了他的剑，他看向她"） | 删掉多余代词 |
| 介词短语开头过长（"关于那件事的一切细节"） | 改为正常中文语序 |
| 定语从句堆叠（"那个穿着红衣服的看起来很生气的女孩"） | 拆分为多个短句 |
| 被动句过多 | 改为主动句 |
| "作为一个X，他..."开头 | 改为更自然的叙述切入 |

### 方法六：保留人味（反过度消毒）

**原则**：改写后的文本必须保留至少一处"只有人类会写"的痕迹。

**审计时可建议作者加入以下"人味元素"：**

| 类型 | 示例 |
|---|---|
| 自嘲/自黑 | "他知道这话挺鸡汤的，但确实这么想的" |
| 承认不确定 | "他记不清了，大概是个冬天？三十岁还是三十一" |
| 不合时宜的小动作 | 说着说着就开始抠指甲/把筷子转来转去 |
| 碎片化的真实思维 | 不是完整的逻辑推理，而是"算了""管他呢""头疼" |
| 私人化的独特表达 | 只有这个角色会说的一句话（口头禅/习惯用语） |

**底线检测**：全文 3000+ 字中如果找不到任何"AI不敢写的话"，建议作者至少加入 1 处。

## 对话质量专项优化（2026年5月新增）

### 问题类型一：对话同质化

**识别特征**：所有角色的说话方式相似，去掉名字后分不清是谁在说。

**优化方案：** 为每个主要角色建立"对话DNA"——

| DNA维度 | 设计要点 |
|---|---|
| 句式偏好 | 话多/话少、长句/短句、喜欢反问/喜欢陈述 |
| 词汇层级 | 文言/白话/网络用语/行业术语 |
| 语气词习惯 | "呢""吧""啊""嘛"的使用频率和位置 |
| 情感表达方式 | 直说/暗示/反话/沉默 |
| 口头禅/习惯用语 | 1-2个独特的小词（"行吧""我看未必""说句不好听的"） |

**改写示例：**

❌ 所有人说话都一样：
```
"我觉得这件事不太对劲。"李明说。
"我也这么认为。"王强说。
"那我们该怎么办？"赵刚说。
```

✅ 区分对话DNA：
```
"不对劲。"李明就说了两个字。
王强想了想："……嗯。"
赵刚急了："那怎么办啊？要不先跑吧？"
```

### 问题类型二：对话功能化（只推进信息，无人物塑造）

**识别特征**：对话内容100%是信息传递，没有废话、没有情绪、没有弦外之音。

**优化方案：** 在信息型对话中插入"人类真实对话"的噪音——

- 打断/抢话
- 答非所问
- 话题偏移
- 重复对方的话（表示在思考）
- 无意义的语气词

❌ 信息型对话：
```
"明天行动开始的时间是早上六点。"
"地点在哪里？"
"城南废弃工厂。"
"明白了。"
```

✅ 加人味：
```
"明天，六点。"他停了一下，"城南那个废工厂。"
"……六点？那不是天还没亮？"
"就是要天没亮。"
她没再问，把地图折起来塞进口袋。
```

### 问题类型三：对话信息密度过高

**识别特征**：角色一口气说超过3句话，每句都在传递新信息，像在念剧本。

**优化方案：** 把长对话拆散，用动作/表情/环境反应隔开。

❌ `他说："我昨天去了那个地方，发现里面有很多奇怪的物品，最中间放着一个箱子，上面刻着一些我看不懂的符号。"`

✅ `他说他昨天去了那个地方。"里面东西挺多，挺怪的。"他顿了顿，"中间有个箱子，上面刻的符号……我看了半天没认出来。"`

## 叙事视角一致性优化（2026年5月新增）

### 问题类型一：视角跳跃（POV错误）

**识别特征**：在限知视角（第三人称限知/第一人称）中，突然写出视角人物不可能知道的信息。

❌ 第三人称限知（跟随主角李明）：
```
李明推开门，房间里没有人。他不知道王强此刻正在三楼的暗室里准备着陷阱。
```
（李明不可能知道王强在准备陷阱——信息越界）

✅ 修正：
```
李明推开门，房间里没有人。
太安静了。安静得让人不放心。
```

### 问题类型二：视角人物知道不该知道的事

**识别特征**：视角人物突然对其他角色的内心活动做出判断。

❌ `李明看着她，她心里很害怕，但她努力掩饰着。`
（如果视角是李明，他只能"看到她努力掩饰"，不能知道"她心里害怕"）

✅ `李明看着她。她脸上没什么表情，但手指在桌子下面攥得很紧。`

### 问题类型三：无预告的视角切换

**识别特征**：在未用分节线（---）或空行分隔的情况下，直接切换视角人物。

**优化方案**：
- 如果是有意切换视角，在切换前加空行或分节线
- 如果是无意切换，统一回主视角人物
