# novel-audit-skill

网文审计 + 去AI化改写一站式技能，适用于所有支持 SKILL.md 规范的 AI 编程助手。

---

## ✨ 核心能力

### 不只是审计 —— 更给出改写方案

本技能在审计的同时，针对每一条问题输出结构化改写建议：

```
❌ 问题原文：
   他心中涌现出一股感动，眼眶微微湿润了。

✅ 优化建议：
   他低下头，用力揉了揉鼻子。

💡 改写思路：
   删除抽象情绪形容词，改为具体细微动作（揉鼻子掩饰情绪），
   避免AI写作常见的"情绪量化+眼眶湿润"套话。
```

每条问题都附带三种信息：**问题定位 → 优化后文本 → 改写思路**，直接可用，无需二次加工。

---

## 功能特性

| 能力 | 说明 |
|------|------|
| **11个审计维度** | 文字质量、AI痕迹、重复雷同、情节合理性、时间线一致性、上下文连贯性、叙事视角一致性、节奏与爽点、人物塑造、世界观自洽、对话质量 |
| **双审计模式** | 单章详细审计 + 多章跨章对比审计 |
| **去AI化改写建议** | 每条问题附带 ❌原文 → ✅建议 → 💡思路，直接可用 |
| **加权综合评分** | A类×0.35 + B类×0.40 + C类×0.25，输出综合评级 |
| **题材全覆盖** | 玄幻/都市/悬疑/言情/历史等，每类有专项检测点 |
| **25项AI快速检测** | 词汇/句式/结构/内容四层检测清单，命中≥5项即预警 |

## 安装

将 `novel-audit` 文件夹放入对应平台的技能目录即可：

| 平台 | 安装路径 |
|------|----------|
| **WorkBuddy** | `~/.workbuddy/skills/novel-audit/` |
| **Cursor** | 项目根目录 `.cursor/skills/novel-audit/` 或全局 `~/.cursor/skills/novel-audit/` |
| **Trae** | 项目根目录 `.trae/skills/novel-audit/` |
| **Windsurf** | 项目根目录 `.windsurf/skills/novel-audit/` |
| **Cline** | 项目根目录 `.cline/skills/novel-audit/` |
| **Roo Code** | 项目根目录 `.roo/skills/novel-audit/` |
| **其他** | 放入平台支持的 skills 目录即可 |

```bash
# 克隆仓库后复制到目标目录
git clone https://github.com/ahao0625/novel-audit-skill.git
cp -r novel-audit-skill/novel-audit <目标平台skills目录>/
```

> **目录结构要求**：安装后须保持 `novel-audit/` 文件夹完整，其中 `SKILL.md`、`references/`、`scripts/` 三个组成部分缺一不可，否则审计时无法加载参考标准。

## 使用方法

在 AI 助手中输入以下触发词即可启动审计：

| 意图 | 触发词示例 |
|------|------------|
| 全文审计 | `审计这一章`、`帮我审稿` |
| AI痕迹检测 | `有没有AI味`、`检查AI痕迹` |
| 连贯性检查 | `前后矛盾吗`、`时间线对吗` |
| 质量评估 | `这章质量怎么样`、`打个分` |

支持直接粘贴章节文本，或指定文件路径进行审计。
支持指定单一维度审计，如：`只检查AI痕迹和对话质量`。

## 📋 审计报告示例

每次审计输出结构化报告，包含：

1. **综合评分** — 各维度0-10分 + 加权总分 + 发布建议
2. **高优问题清单** — 🔴 必须修改的问题，每条附改写方案
3. **中优问题清单** — 🟡 建议修改的问题
4. **亮点记录** — ✅ 写得好的段落，保留风格
5. **优先修改建议** — 按优先级排序的修改清单

## 11个审计维度

| 类别 | 权重 | 维度 | 核心检测点 |
|------|------|------|------------|
| **A类 基础质量** | ×0.35 | A1 文字质量 | 病句、流水账、形容词堆砌、冗余表达 |
| | | A2 重复雷同 | 高频词、段内同义重复、情节结构克隆 |
| | | A3 AI痕迹 | 7类套话词库 + 6大句式模板 + 25项检测清单 |
| **B类 叙事逻辑** | ×0.40 | B1 情节合理性 | 动机充分性、因果链、上帝之手 |
| | | B2 时间线一致性 | 时序矛盾、年龄/季节一致、故事内时钟 |
| | | B3 上下文连贯性 | 设定连贯、人物状态承接、情绪承接 |
| | | B4 叙事视角一致性 | POV越界、读取他人内心、无预告视角切换 |
| **C类 阅读体验** | ×0.25 | C1 节奏与爽点 | 拖沓识别、爽点密度、章末钩子、开篇吸引力 |
| | | C2 人物塑造 | 性格一致性、对话口吻辨识度、成长弧 |
| | | C3 世界观自洽 | 规则一致性、力量代价、势力关系 |
| | | C4 对话质量 | 对话同质化、功能化、信息密度、弦外之音 |

> **权重说明**：B类（叙事逻辑）权重最高，因为情节BUG和时间线矛盾是影响阅读体验的核心问题。

## 去AI化改写方案

除审计外，本技能内置完整的**去AI化改写方案库**，覆盖：

### 六法主动优化

| 方法 | 说明 |
|------|------|
| **动词强化** | 把"进行/实现/发挥"等抽象万能动词替换为具体动作 |
| **节奏重塑** | 打破AI的"机关枪节奏"，长短句错落有致 |
| **Filler切除** | 删掉"在一定程度上/某种意义上"等不承载信息的包装词 |
| **抽象换具体** | 把"繁华的街道"换成"卖糖葫芦的吆喝声盖过了车马声" |
| **语序归位** | 修复翻译腔残留（代词泛滥/被动句过多/英文成语直译） |
| **保留人味** | 加入自嘲、不确定、不合时宜的小动作等"AI不敢写"的细节 |

### 对话质量专项优化

- 为每主要角色建立**对话DNA**（句式偏好/词汇层级/语气词/口头禅）
- 消除对话同质化（去掉名字分不清是谁在说话）
- 降低对话信息密度（用动作/表情隔开长对话）
- 加入弦外之音（30%的对话含隐藏动机）

## 评分与发布建议

| 综合分 | 评级 | 发布建议 |
|--------|------|----------|
| 8.5–10 | 🟢 优质 | 可直接发布，锦上添花式修改 |
| 7.0–8.4 | 🟡 良好 | 处理高优问题后可发布 |
| 5.5–6.9 | 🟠 及格 | 处理高优+中优问题后发布 |
| 4.0–5.4 | 🔴 待改进 | 需要较多修改，建议暂缓发布 |
| 0–3.9 | ⚫ 需大改 | 建议重写或大幅修改 |
