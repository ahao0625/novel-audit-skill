# 网文审计技能 (Novel Audit)

## 触发条件

以下情况触发本技能：

- 用户粘贴网文文本并要求检查/审计/看看质量
- 用户问"这章有没有问题"、"前后矛盾吗"、"AI味重不重"
- 用户明确说"帮我审稿"、"全面检查一下"
- 用户指定单个维度："只看AI痕迹"、"检查时间线"、"有没有重复"

## 工作流程

### 第一步：判断输入模式

- **单章模式**：输入为1章或1段，执行全量9维度详细审计
- **多章模式**：输入为2章或以上，在9维度基础上额外执行跨章一致性对比
- **指定维度模式**：用户只要求审计特定维度时，聚焦执行对应维度，其余维度跳过

### 第二步：读取参考文档

审计前，加载以下参考文档作为判断依据：

- `references/audit-criteria.md`：9个维度的详细评判标准和典型反例
- `references/ai-patterns.md`：AI写作特征词库和句式模板库
- `references/scoring-rubric.md`：0-10分各档位描述标准
- `references/optimization-guide.md`：各维度问题的具体优化方案和改写示例，审计时必须参考此文档给出可操作的修改建议

### 第三步：识别题材类型（可选）

根据文本内容自动识别题材（玄幻/都市/悬疑/言情/历史等），激活对应的题材专项检查点（详见 `references/audit-criteria.md` 第四节）。若题材不明或属于通用叙事，跳过题材专项检查。

### 第四步：逐维度审计

按以下顺序执行审计，每个维度独立输出评分和问题列表：

```
A. 文本质量类
  A1 文字基础质量
  A2 重复雷同检测
  A3 AI痕迹识别（含去AI化改写建议）

B. 叙事逻辑类
  B1 情节合理性
  B2 时间线一致性
  B3 上下文连贯性
  B4 叙事视角一致性（POV检测）

C. 读者体验类
  C1 节奏与爽点设计（含开篇吸引力/信息密度）
  C2 人物塑造一致性
  C3 世界观自洽性
  C4 对话质量专项
```

> 说明：B4 检测视角越界、无预告切换；C4 检测对话同质化、信息密度、弦外之音。

### 第五步：输出审计报告

严格按照下方报告模板输出，不得省略评分表和问题清单。

## 使用说明

### 基本用法

```
用户粘贴文本后说：
"帮我审计这章"  →  全量9维度审计
"只看AI痕迹"    →  仅执行A3维度
"检查时间线"    →  仅执行B2维度
"审计这两章，看看前后是否连贯"  →  多章对比模式
```

### 输出规范

- 所有问题必须引用原文具体句子，不得泛泛而谈
- 评分需有依据，不得无理由给高分或低分
- 去AI化/优化方案必须提供改写后的具体文本，不只是说"改掉"或"加强描写"
- 优先修改建议必须分级（红/黄/绿），且条数合理（3-8条）
- 每个维度下的"优化方案"至少给出1条具体改写示例（用 ❌/✅ 格式），问题较多时可给出多条
- 若文本整体质量较好、问题较少，优化方案可合并为"整体提升建议"段落，仍须给出具体改写示例

### 边界情况处理

- **文本太短（<500字）**：仍执行全量审计，但说明"样本量较小，部分维度仅供参考"
- **多章文本太长（>10000字）**：先做整体扫描，再对关键段落做精细审计
- **用户提供了故事背景/人设**：以用户提供的背景为基准，检查文本是否与之一致
- **文本质量整体较高**：仍需列出问题，但可适当减少低优条目，重点给出提升建议

---

## 注意事项

- 审计是辅助，不是否定。发现优点时也要在总评中指出
- AI痕迹判断要有依据，避免误判（不是所有排比句都是AI写的）
- 世界观/时间线问题需要有原文证据，不能凭空推断
- 对于"刻意为之"的写法（如某角色说话就是简短粗鲁），不应判定为问题
